
# SQLGuard - CTF注入语法自动提示与自动化测试工具

## 1. 项目介绍

SQLGuard是一款专为CTF比赛设计的SQL注入测试工具，提供自动化的注入点检测和注入语法生成。该工具通过本地代理服务器绕过浏览器的CORS限制，实现跨域请求测试，帮助CTF选手快速定位和利用SQL注入漏洞。

![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/160fb7f3ac9d46dfa89f32bc7f4df14e.png)


### 主要功能
- 支持GET和POST请求方法的注入测试
- 自动化注入点检测与验证
- 多类型注入语法自动生成
- 详细的测试结果与统计信息
- 支持多种注入技术（布尔盲注、时间盲注、报错注入等）
- 灵活的测试选项配置
- 结果导出功能（Excel格式）
- 支持多种数据库类型的时间盲注

## 2. 环境要求

- **Node.js 12+**：用于运行本地代理服务器
- **现代浏览器**：Chrome 80+、Firefox 75+、Edge 80+、Safari 13+

如果您尚未安装Node.js，请按照以下步骤安装：

1. 访问Node.js官网：https://nodejs.org/
2. 下载并安装适合您操作系统的Node.js版本（推荐LTS版本）

### 3.2 启动本地代理服务器

**这是必须的第一步！** 现代浏览器有严格的CORS策略，直接从本地文件运行的网页无法向其他域名发送AJAX请求。本地代理服务器作为中间层，绕过了这一限制。

1. 打开命令行终端
2. 进入SQLGuard项目目录：
   ```bash
   cd SQLGuard
   ```
3. 执行以下命令启动代理服务器：
   ```bash
   node proxy.js
   ```
4. 成功启动后，终端将显示：
   ```
   Proxy server running at http://localhost:3000
   Usage: http://localhost:3000?url=YOUR_TARGET_URL
   ```

## 5. 功能详解

### 5.1 自动化测试

点击「开始自动化测试」按钮后，工具将执行以下操作：

1. 根据选择的注入类型和测试深度生成测试用例
2. 通过本地代理服务器发送请求，绕过CORS限制
3. 分析响应结果，检测注入点
4. 实时显示测试结果和统计信息

### 5.2 测试选项说明

#### 5.2.1 注释变体

SQL注入中常用的注释符号，用于注释掉SQL语句的剩余部分：
- 不带注释
- 双横线注释 `--`
- 双横线+注释 `-- `
- 井号注释 `#`
- 多行注释 `/* ... */`
- 内联注释 `//`
- 双重注释

#### 5.2.2 字段包裹符

SQL语句中字段可能被不同符号包裹：
- 无
- 单引号 `'`
- 双引号 `"`
- 反引号 ```````
- 方括号 `[]`
- 小括号 `()`
- 单引号+小括号 `'()`
- 等多种组合

#### 5.2.3 空格替代

在某些情况下，空格可能被过滤，需要使用替代字符：
- 无（正常空格）
- Tab制表符 `\t`
- 换行符 `\n`
- 注释空格 `/* */`
- 加号连接 `+`

#### 5.2.4 逻辑运算符

用于构建注入测试条件：
- 无
- AND
- OR
- AND变体 `&&`
- OR变体 `||`
- 不等于 `<>`
- LIKE
- IN
- REGEXP

#### 5.2.5 时间盲注

针对不同数据库的时间盲注技术：
- 无
- 时间盲注MySQL
- 时间盲注PostgreSQL
- 时间盲注SQLServer

### 5.3 结果分析

- **URL**：测试的完整URL

#### 5.6.4 注入尝试操作详解

尝试注入功能生成的注入尝试操作包括：

| 操作名称 | 描述 | 示例Payload | 预期结果 |
|---------|------|-------------|----------|
| 测试字段输出位置 | 检测哪些字段会在页面中显示 | `' union select 1,2,3 --+` | 页面中显示数字2或3，表示对应的字段会在页面中输出 |
| 获取数据库版本 | 提取数据库版本信息 | `' union select version(),2,3 --+` | 页面中显示数据库版本号，如 "5.7.31" |
| 获取当前数据库名 | 提取当前使用的数据库名称 | `' union select database(),2,3 --+` | 页面中显示当前数据库名称，如 "testdb" |
| 获取当前用户 | 提取当前数据库用户 | `' union select user(),2,3 --+` | 页面中显示当前数据库用户，如 "root@localhost" |
| 获取数据库列表 | 提取所有数据库名称 | `' union select schema_name,2,3 from information_schema.schemata --+` | 页面中显示数据库列表，如 "information_schema", "testdb" |
| 获取表列表 | 提取指定数据库中的所有表名 | `' union select table_name,2,3 from information_schema.tables where table_schema=database() --+` | 页面中显示当前数据库中的表列表，如 "users", "products" |
| 获取列列表 | 提取指定表中的所有列名 | `' union select column_name,2,3 from information_schema.columns where table_name='users' --+` | 页面中显示users表中的列列表，如 "id", "username", "password" |
| 获取数据 | 提取指定表中的数据 | `' union select username,password,3 from users --+` | 页面中显示users表中的数据，如 "admin", "123456" |

#### 5.6.5 高级使用技巧

1. **确定字段数**：
   - 使用「测试字段输出位置」操作，逐步调整字段数
   - 例如，从 `' union select 1 --+` 开始，逐步增加数字，直到页面正常显示
   - 当页面显示异常时，前一个数字即为正确的字段数

2. **处理不同的注入类型**：
   - 对于数字型注入，Payload无需引号包裹：`order by 2 --+`
   - 对于字符型注入，Payload需要引号包裹：`' order by 2 --+`
   - 对于不同的引号类型，需要调整Payload格式

3. **利用错误信息**：
   - 观察注入结果中的错误信息，了解数据库类型和结构
   - 根据错误信息调整注入策略

4. **组合使用多种注入技术**：
   - 结合布尔盲注和时间盲注技术
   - 使用不同的注释变体和空格替代
   - 尝试多种逻辑运算符

#### 5.6.6 常见问题与解决方案

**Q: 注入尝试执行失败怎么办？**
A: 
- 检查代理服务器是否正常运行
- 确认目标URL是否可访问
- 调整Payload格式，尝试不同的注释变体和空格替代
- 降低并发数，增加超时时间

**Q: 注入结果显示正常，但没有提取到数据？**
A: 
- 检查字段数设置是否正确
- 确认注入点的字段位置
- 尝试不同的Payload格式
- 检查目标网站是否有输出限制

**Q: 如何处理大量数据的情况？**
A: 
- 使用LIMIT子句分页获取数据
- 结合条件查询，缩小数据范围
- 使用GROUP_CONCAT函数合并数据

#### 5.6.7 注意事项

- 尝试注入功能需要在测试结果的基础上使用，因此需要先完成自动化测试
- 生成的注入尝试操作可能需要根据实际情况进行调整
- 某些注入尝试可能会被目标网站的防护机制拦截
- 在使用尝试注入功能时，建议逐步执行，观察每个操作的结果
- 对于CTF题目，尝试注入功能可以帮助快速获取flag
- 仅用于合法的网络安全测试和学习，请勿用于非法用途

## 6. 常见问题与解决方案

### 6.1 代理服务器相关问题

**Q: 为什么必须启动本地代理服务器？**
A: 现代浏览器有严格的CORS策略，直接从本地文件运行的网页无法向其他域名发送AJAX请求。本地代理服务器作为中间层，绕过了这一限制。

**Q: 如何判断代理服务器是否正常运行？**
A: 启动代理服务器后，终端会显示服务器运行信息。网页顶部会显示代理服务器的状态：
- ✅ 代理服务器运行正常
- ❌ 代理服务器未运行，请先执行 `node proxy.js`

**Q: 代理服务器启动失败怎么办？**
A: 
1. 检查端口3000是否被占用：执行 `netstat -ano | findstr :3000`（Windows）或 `lsof -i :3000`（Linux/macOS）
2. 如果端口被占用，可以修改 `proxy.js` 文件中的端口号
3. 确保Node.js已正确安装：执行 `node -v` 检查

### 6.2 测试相关问题

**Q: 测试时出现"CORS跨域限制"错误怎么办？**
A: 请确保本地代理服务器已正确启动。如果问题仍然存在，可能是目标服务器的CORS策略过于严格，建议使用「仅生成注入语法」功能，直接在浏览器中访问生成的URL。

**Q: 测试时出现"连接被重置"错误怎么办？**
A: 这通常是因为服务器对频繁的请求进行防护。建议：
1. 降低并发数（1-3）
2. 增加超时时间（10-20秒）
3. 减少测试选项
4. 使用「仅生成注入语法」功能手动测试

**Q: 测试结果显示"无注入点"，但我确定存在注入漏洞怎么办？**
A: 
1. 检查测试选项配置是否正确
2. 尝试调整测试选项，如增加注释变体、字段包裹符等
3. 使用「仅生成注入语法」功能，手动测试生成的注入语法
4. 检查目标URL是否正确，是否需要登录验证

**Q: 测试速度很慢怎么办？**
A: 
1. 降低测试深度
2. 减少测试选项
3. 增加并发数（注意不要超过10，避免对目标服务器造成过大压力）
4. 减少测试参数数量
