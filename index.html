SQLGuard 使用说明
1. 工具简介

SQLGuard 是一款 CTF 注入语法自动提示与自动化测试工具，用于检测和测试 SQL 注入漏洞。

2. 基本使用流程
步骤 1：配置目标信息

请求方法：选择 GET 或 POST 请求方法
目标 URL：输入要测试的目标 URL，如 https://sqli.example.com/Less-11/
解析 URL：点击「解析」按钮可自动提取 URL 中的参数

步骤 2：管理测试参数

参数列表：显示所有请求参数，包括 URL 参数和 POST 参数
注入点勾选：勾选要作为注入点测试的参数
添加/删除参数：使用「添加参数」和「删除参数」按钮管理参数

步骤 3：设置测试选项

注释变体：选择要测试的 SQL 注释类型
字段包裹符：选择可能的字段包裹符（如单引号、双引号等）
空格替代：选择可能的空格替代方式
逻辑运算符：选择要测试的逻辑运算符
时间盲注：选择是否使用时间盲注技术
并发数：设置并发测试数量
超时时间：设置请求超时时间

步骤 4：开始测试

开始自动化测试：点击按钮启动自动化 SQL 注入检测
仅生成注入语法：仅生成可能的注入语法，不执行实际测试
清空结果：清空当前测试结果

3. 测试结果查看
测试统计

显示总测试数、注入点发现数量、可疑注入点数量和测试错误数量

结果筛选

结果类型筛选：筛选注入点发现、可疑注入点、无注入点或测试错误
风险等级筛选：按高风险、中风险或低风险筛选
关键词搜索：搜索特定关键词

结果列表

排序功能：点击表头可对结果进行排序
列筛选：在筛选行中输入筛选条件
导出Excel：将测试结果导出为Excel文件
分页查看：使用分页控件浏览大量结果

4. 高级功能：尝试注入

在测试结果列表中，点击「尝试注入」按钮可以打开注入尝试窗口，进行更深入的注入测试：

4.1 确定字段数

通过 ORDER BY 语句确定表中的字段数量

4.2 猜测显示位置

通过 UNION SELECT 语句猜测注入结果的显示位置

4.3 执行注入

联合查询注入：使用 UNION SELECT 语句获取数据
双查询注入：使用嵌套查询获取数据
基于错误的XPATH注入：使用 updatexml 或 extractvalue 函数获取数据

5. 配置管理

您可以保存当前的测试配置，方便后续重复使用：

5.1 保存配置

输入配置名称，点击「保存配置」按钮保存当前配置

5.2 加载配置

在保存的配置列表中，点击配置项即可加载该配置

6. 注意事项
  • 仅用于合法的网络安全测试和学习，请勿用于非法用途
  • 在实际渗透测试中，应遵守相关法律法规，仅在授权范围内测试
  • 对于CTF题目，重点关注工具生成的注入语法，结合手动测试
  • 定期更新工具，以获取最新的注入技术支持
